# Extension Issues & Improvements

> Collected: 2026-03-01
> Source: manual testing after FEAT-001 implementation

## Bugs

### BUG-1: "No target tab found" при нажатии "Начать звонок"

**Серьёзность:** Critical — звонок не начинается вообще

**Где:** `extension/src/background/service-worker.ts:76-82`

**Причина:** Popup отправляет `START_SESSION` через `chrome.runtime.sendMessage()`, но service worker берёт `sender.tab?.id` для `tabCapture.getMediaStreamId()`. Popup — не вкладка, поэтому `sender.tab` всегда `undefined`.

**Как исправить:** Popup должен определить активную вкладку через `chrome.tabs.query({ active: true, currentWindow: true })` и передать `tabId` в сообщении. Service worker использует переданный `tabId` вместо `sender.tab?.id`.

**Файлы:**
- `extension/src/popup/popup.ts:256-277` — добавить получение активной вкладки
- `extension/src/shared/messages.ts` — добавить `tabId` в тип `START_SESSION`
- `extension/src/background/service-worker.ts:76` — использовать `message.tabId` вместо `sender.tab?.id`

---

### BUG-2: Настройки URL бэкенда не влияют на реальные запросы

**Серьёзность:** Medium

**Где:** `extension/src/popup/popup.ts:363-386` и `extension/src/shared/constants.ts`

**Причина:** Настройки URL сохраняются в `chrome.storage.local`, но все запросы используют захардкоженный `API_BASE` из `constants.ts` (`http://localhost:8000/api/v1`). Сохранённые настройки никуда не читаются при выполнении fetch-запросов.

**Как исправить:** Либо убрать поле URL из настроек (оно вводит в заблуждение), либо сделать чтение URL из storage перед каждым запросом.

## UI/UX Improvements

### UI-1: Редизайн навигации — убрать вкладку "Файлы", объединить с "Брифинг"

**Сейчас:** 3 вкладки: Файлы | Брифинг | Настройки. Вкладка "Файлы" содержит загрузку + управление звонком. Вкладка "Брифинг" отдельная.

**Новый дизайн:**
- **Убрать вкладку "Файлы"** полностью
- **Вкладка "Брифинг"** (основная, по умолчанию): загрузка файлов сверху → текст брифинга снизу (автоматически после загрузки)
- **Вкладка "Настройки"**: все технические настройки
- **Кнопка REC** — одна главная кнопка, всегда видна:
  - Выключен (по умолчанию): серый цвет, текст "REC" или "Копайлот"
  - Включён: **горит красным** (как на студиях звукозаписи), пульсирует
  - Заменяет пару кнопок "Начать звонок" / "Завершить"

### UI-2: Брифинг показывает сырой JSON вместо читаемого текста

**Сейчас:** Портрет покупателя и Стратегия отображаются как сырой текст из JSON — `role: Коммерческий директор, pain_points: [ ... ], motivators: [ ... ]`. Выглядит как дамп данных. Всё слипается в сплошную стену текста.

**Предложение:** Отформатировать каждую секцию брифинга с визуальным разделением и выделением:

**Портрет покупателя:**
- Роль и имя — крупным жирным заголовком
- Боли (pain_points) — маркированный список с иконкой (красный кружок/!), каждая боль отдельной строкой
- Мотиваторы — теги/чипы (бейджи) с акцентным цветом
- Бюджет — отдельная строка с иконкой
- Стиль общения — курсивом или в отдельном блоке-цитате

**Стратегия:**
- Подход — выделенный абзац (highlight box / цветной фон)
- Ключевые сообщения — нумерованный список, каждое сообщение визуально отделено

**Возражения:**
- Каждое возражение — отдельная карточка/аккордеон
- Возражение жирным красным → ответ обычным текстом
- Визуальное разделение между возражениями (divider или отступ)

**Общее:** Использовать whitespace, разделители, разные размеры шрифтов для визуальной иерархии. Ключевые моменты выделять жирным или цветом.

### UI-8: Content-script виджет ("AI слушает") — нет анимации при активном аудио

**Сейчас:** Внизу страницы CRM появляется виджет-полоса "AI слушает" (content script `widget.ts`). Но визуально непонятно, идёт ли реально захват аудио или нет. Виджет статичен — нет анимации звуковой волны, нет пульсации, ничего.

**Предложение:**
- Добавить анимацию звуковой волны (waveform / equalizer bars) когда реально поступает аудио сигнал
- Без аудио — виджет показывает статичное состояние "ожидание"
- При получении транскрипта — кратковременная подсветка
- При получении подсказки — визуальное выделение (glow / bounce)

---

### UI-9: Ключевая проблема — Copilot не работает end-to-end

**Сейчас:** Основной сценарий (загрузка файлов → начало звонка → аудио захват → транскрипция → подсказки) не работает из-за цепочки проблем:
1. BUG-1: кнопка "Начать звонок" падает с "No target tab found"
2. Даже если починить — нет аудио потока (tabCapture или mic не инициализируется)
3. Без аудио нет транскрипции → нет подсказок → copilot бесполезен

**Это блокер.** Без рабочего аудио pipeline весь продукт — это просто генератор брифингов.

## Technical Debt

### TECH-1: Popup CSS не оптимизирован для маленького окна

Popup расширения имеет фиксированный размер ~400x600px. Некоторые элементы могут не помещаться.

---

### TECH-2: Нет обработки потери соединения WebSocket

Если WebSocket обрывается, нет автоматического переподключения. Пользователь не узнает о потере связи.

---

### TECH-3: Offscreen document lifecycle

Offscreen document создаётся при старте сессии, но может быть убит Chrome в любой момент. Нужна проверка и пересоздание.

---
