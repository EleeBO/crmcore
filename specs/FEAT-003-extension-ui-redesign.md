# FEAT-003: Extension UI Redesign & Polish

Status: DRAFT
Created: 2026-03-01
Last Modified: 2026-03-01

## Overview

Редизайн UI Chrome-расширения: убрать вкладку "Файлы" (объединить с "Брифинг"), одна кнопка REC (красная/серая как на студии), отформатировать брифинг из сырого JSON в читаемый текст, добавить анимацию аудио в content-script виджете. Зависит от FEAT-002 (рабочий audio pipeline).

## Dependencies

- **FEAT-002** (Audio Pipeline Fix) — must be ACTIVE before starting this feature

## Acceptance Criteria

### AC-1: Popup имеет 2 вкладки: Брифинг (основная) и Настройки
- Given пользователь открыл popup
- Then вкладка "Файлы" отсутствует
- And на вкладке "Брифинг" сверху — зона загрузки файлов, снизу — отформатированный брифинг

### AC-2: Одна кнопка REC
- Given файлы загружены
- When пользователь нажимает REC (серая кнопка)
- Then кнопка загорается красным, пульсирует (как на студии звукозаписи)
- When нажимает снова
- Then кнопка гаснет (серая), сессия останавливается

### AC-3: Брифинг отформатирован красиво
- Given сценарий сгенерирован
- Then портрет покупателя показан структурировано (роль жирным, боли списком, мотиваторы чипами)
- And стратегия с выделенными ключевыми сообщениями
- And возражения как отдельные карточки с визуальным разделением
- And секции визуально разделены, ключевые моменты выделены

### AC-4: Виджет анимирован при активной сессии
- Given сессия запущена (REC горит красным)
- Then виджет на странице CRM показывает CSS-анимацию (equalizer bars)
- When приходит транскрипт — кратковременная подсветка
- When сессия неактивна — виджет статичен

### AC-5: Автоматический показ брифинга после загрузки
- Given пользователь загрузил файлы
- When загрузка завершена и сценарий сгенерирован
- Then брифинг автоматически отображается (без нажатия кнопки)
